# ESP32-C3 并联腿机器人控制 + IMU 姿态自平衡案例

这是一个基于 ESP32-C3 开发板与 PlatformIO 搭建的 **双自由度 (2-DOF) 并联腿机器人** 运动控制与 IMU 姿态自平衡案例。

并联腿（如经典的五杆闭链机构，Five-bar linkage）由于其电机（舵机）都可以安装在靠近身体的根部，具有极低的腿部惯量，非常适合制作高性能的双足或四足跑跳机器人。

此外，本项目集成了 **MPU6050 IMU** 传感器，通过互补滤波算法计算出机身姿态（Pitch），实现了闭环姿态反馈调节（自平衡模式）。

---

## 1. 机构几何模型与原理

该并联腿使用五杆闭链机构，其坐标系与结构定义如下：

```
       [ 机器人身体 (y = 0) ]
     A1 (-d, 0)      A2 (d, 0)   <-- 左右两个主动舵机轴心
       \            /
   L1   \          /   L1        <-- 主动臂 (Link 1)
         \        /
         B1      B2              <-- 肘部转轴 (无源关节)
         /        \
   L2   /          \   L2        <-- 从动臂 (Link 2)
       /            \
       \            /
        \          /
         \        /
          \      /
           \    /
            \  /
             \/
             P (x, y)            <-- 足端 (Foot End-effector)
```

- **原点 (0, 0)**: 左右舵机旋转轴心连线的中点。
- **$A_1, A_2$**: 左右舵机轴心位置，坐标分别为 $(-d, 0)$ 和 $(d, 0)$。
- **$L_1$**: 主动臂长度 (从舵机臂到肘部)。
- **$L_2$**: 从动臂长度 (从肘部到足端)。
- **$B_1, B_2$**: 左右肘部关节坐标。
- **$P(x, y)$**: 足端坐标。

### 逆运动学 (Inverse Kinematics, IK)
通过给定的足端目标位置 $P(x, y)$，反算出左右舵机的目标控制角度 $\theta_1$ 和 $\theta_2$。
- 利用余弦定理和极坐标变换求解三角形 $A_1B_1P$ 和 $A_2B_2P$。
- 设定肘部向外弯曲（肘朝外，Outer-bending），以获得最大的工作空间并避免杆件相撞。

### 顺运动学 (Forward Kinematics, FK)
通过给定的左右舵机角度 $\theta_1$ 和 $\theta_2$，计算出当前足端所处的坐标 $P(x, y)$。
- 求解两圆相交问题：圆心分别为 $B_1$ 和 $B_2$，半径均为 $L_2$ 的两个圆的交点中，取 $Y$ 值较小的交点（向下延伸的点）。

## 2. 硬件引脚分配 (ESP32-C3)

本案例使用 **ESP32-C3** 作为中央控制器，通过 GPIO 矩阵复用一个硬件串口分别发送指令至 6 个 YYT 闭环电机驱动板，同时接入 I2C IMU（MPU6050）传感器。

根据部分 ESP32-C3 开发板的丝印规范（如合宙、LuatOS 等经典板型，其 `IOxx` 丝印很容易被误读为 `10xx`，如 `IO05` 读作 `1005`，`IO08` 读作 `1008`），下表同时列出了 **标准 GPIO 编号** 与 **开发板可能对应的丝印标注**：

| 外设 / 传感器 / 电机 | ESP32-C3 GPIO 编号 | 常见开发板丝印标注<br>(可能显示为 10xx / IOxx) | 逻辑电平 | 连接端说明 | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **IMU SDA** | `GPIO8` | **1008** / `IO08` | 3.3V | MPU6050 SDA | I2C 数据线，需外接 4.7kΩ 上拉电阻 |
| **IMU SCL** | `GPIO9` | **1009** / `IO09` | 3.3V | MPU6050 SCL | I2C 时钟线，需外接 4.7kΩ 上拉电阻 |
| **左腿电机 1 (L1)** | `GPIO0` | **1000** / `IO00` | 3.3V | 左 1 驱动板 RX (STM32 PB7) | 主动关节 1 |
| **左腿电机 2 (L2)** | `GPIO1` | **1001** / `IO01` | 3.3V | 左 2 驱动板 RX (STM32 PB7) | 主动关节 2 |
| **左腿辅助电机 (L3)** | `GPIO6` | **1006** / `IO06` | 3.3V | 左 3 驱动板 RX (STM32 PB7) | 辅助关节 / 拓展 |
| **右腿电机 1 (R1)** | `GPIO3` | **1003** / `IO03` | 3.3V | 右 1 驱动板 RX (STM32 PB7) | 主动关节 1 |
| **右腿电机 2 (R2)** | `GPIO4` | **1004** / `IO04` | 3.3V | 右 2 驱动板 RX (STM32 PB7) | 主动关节 2 |
| **右腿辅助电机 (R3)** | `GPIO5` | **1005** / `IO05` | 3.3V | 右 3 驱动板 RX (STM32 PB7) | 辅助关节 / 拓展 |
| **指示 LED 灯** | `GPIO10` | **1010** / `IO10` | 3.3V | 板载 / 外部指示灯正极 | 呼吸灯指示系统状态 |
| **串口调试 (USB Serial)** | Native USB (GPIO18/19) | **1018/1019** / `IO18/19` | - | PC 电脑端 | 调试与 VOFA+ 监控 (115200 bps) |

> **[!WARNING]**
> * **启动引导引脚 (Strapping Pins) 规避**：ESP32-C3 的 `GPIO2`, `GPIO8`, `GPIO9` 为上电启动引脚。为了保证开发板正常开机，我们在电机发送控制线上避开了这几个引脚，防止在系统刚上电时因驱动板拉低电平导致 C3 启动失败。
> * **IMU 引脚与启动**：虽然 IMU 占用了启动引脚 `GPIO8` (`1008`) 和 `GPIO9` (`1009`)，但因为 I2C 总线两端连接了 **4.7kΩ 的上拉电阻**（通常集成在 MPU6050 模块上），上电时会自动将引脚拉高，这刚好符合 ESP32-C3 正常从 Flash 启动所需的电平状态，因此可以安全使用。

## 2.1 YYT 电机闭环接入指南

为了让 ESP32-C3 能够顺畅驱动并联腿的 6 个 YYT 闭环轮毂/伺服电机，请严格按照以下电气与通信规范进行配置和接入：

### ① 电气接线规范
* **信号线直接物理对连**：ESP32-C3 的 `GPIO0/1/3/4/5/6` 发送引脚与各驱动板上的 RX (`PB7` 引脚) 采用单线直连。由于两端的 I/O 口逻辑电压均为 **3.3V**，不需要任何电平转换板。
* **信号地共地**：必须将 ESP32-C3 开发板的 **GND** 和所有 6 个 YYT 电机驱动板的 **GND** 用粗导线连接在一起。
* **大电流动力供电**：
  * **严禁**使用 ESP32-C3 的 `5V` 或 `3.3V` 输出引脚为电机或驱动板的动力输入供电。
  * 驱动板的动力输入端（如 `VBUS/GND`）应外接独立的 **12V~24V** 高放电倍率动力电池或大功率稳压电源。

### ② 下位机（YYT 驱动板）配置步骤
1. **烧录对应 ID 的固件**：
   * 打开 `YYT_MiniOdrive` 目录，或者直接烧录 `firmware_ids/turing_CBU6_idX.bin` 固件。
   * 虽然由于硬件上采用了一对一的 GPIO 单线连接，总线数据天然隔离，但建议在下位机中配置相应的关节 ID，以便后续通过串口总线回传数据或进行针对性零偏校准。
2. **磁编码器校准**：
   * 电机组装好后，根据 `闭环参数调试.docx`，使用上位机工具 VOFA+ 监控并校准 AS5600 或 MT6816 磁传感器，确保零点校准保存到 STM32 的 Flash 中。

### ③ 核心控制通信协议
C3 与驱动板之间的通信使用简易 ASCII 协议，波特率为 **115200, 8N1**。常用的控制指令包括：

* **模式配置指令 (`M0`)**：
  * 格式：`M0=<mode_val>!` 
  * 说明：在系统初始化时，C3 会循环发送 `M0=0!` 到各个电机，将其配置为**位置闭环控制模式**。
* **目标位置指令 (`P0`)**：
  * 格式：`P0=<angle_in_radians>!`
  * 说明：C3 根据并联腿运动学算出的各关节目标弧度值，实时下发此指令（例如 `P0=1.5708!` 控制电机转动至 $\pi/2$ 弧度位置）。

## 4. IMU 姿态解算与自平衡控制

1. **I2C 设备扫描**: 
   启动时，主机会自动扫描 I2C 总线并打印所有在线设备。这方便了用户排查接线故障，确保 IMU（如 MPU6050 默认地址 `0x68`）正确挂载。
2. **互补滤波姿态估计 (Complementary Filter)**:
   - 融合陀螺仪的积分角度与加速度计得到的重力夹角。
   - 表达式：$Angle = 0.96 \times (Angle + GyroRate \times dt) + 0.04 \times AccelAngle$。
   - 具有极高的稳定性和响应速度，有效滤除高频电机振动干扰。
3. **姿态平衡环控制 (MODE_BALANCE)**:
   - 闭环控制算法：机身向前倾斜（$Pitch > 0$）时，控制脚掌向前偏移以维持中心平衡；
   - 控制率：$X_{\text{compensate}} = Pitch \times K_p$ ($K_p = -1.2 \text{mm/deg}$)。
   - 可在开发板倾斜时实时驱动舵机运动以使足端朝反方向平移支撑。

## 5. 串口指令控制与交互

烧录程序后，通过串口助手（如 PlatformIO Serial Monitor，波特率 `115200`）连接机器人，可发送以下指令进行交互：

- `standby`: 机器人腿部将收回并静态悬停在中位 `(0, -100)` mm 处（开机默认模式）。
- `balance`: 开启 IMU 自平衡反馈（当倾斜机身时，机器人会根据 MPU6050 采集到的 Pitch 角度反向移动以维持平衡）。
- `X Y`: 手动控制足端移动到指定坐标（例如输入 `10 -90`，回车后足端会平滑移动到该位置。若输入的值超出了可达工作空间，终端会输出警告且保持安全限幅）。

## 7. 系统通信架构与协议规范

系统主要由三部分组成：**ESP32-C3（主控）**、**YYT MiniOdrive（电机驱动器）** 和 **PioPulse（PC 端 TUI 可视化软件）**。它们之间的通信机制如下：

### ① ESP32-C3 $\leftrightarrow$ YYT 电机驱动器
* **物理连接**：ESP32-C3 通过内置的 GPIO 矩阵特性（`uart_set_pin`），将单路硬件串口 `UART1` 的 TX 信号根据当前寻址目标实时切换输出到不同电机的 GPIO（`GPIO 0/1/6/3/4/5`），分别物理直连至各驱动板的 RX（STM32 `PB7`）。
* **串口配置**：波特率 **`115200`**，数据位 **`8`**，停止位 **`1`**，无校验。
* **ASCII 控制协议**：
  * **模式配置** (`M0`)：`M0=<mode_val>!` （如 `M0=0!` 为位置闭环控制模式）。
  * **位置目标** (`P0`)：`P0=<rad_val>!` （如 `P0=1.5708!` 控制电机转动到 $\pi/2$ 弧度位置）。

### ② ESP32-C3 $\leftrightarrow$ PioPulse 终端可视化
* **物理连接**：ESP32-C3 的 Native USB/调试串口（`GPIO 18/19`）通过 USB 线连接至 PC 电脑。
* **串口配置**：波特率 **`115200`**，流控无。
* **TUI 帧传输协议 (FireWater)**：
  * **数据格式**：以 `\n`（换行符）结尾的逗号分割 ASCII 字符串。
  * **输出示例**：`imu:pitch_val,roll_val,yaw_val,0.0,0.0,0.0\n` （前三位为 MPU6050 经过互补滤波后的 Pitch, Roll, Yaw 姿态度数；后三位为平移占位符，PioPulse 内置的 3D 立方体控件将会直接读取这 6 个通道渲染出实时 3D 姿态）。
